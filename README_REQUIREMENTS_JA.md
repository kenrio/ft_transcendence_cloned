\*\*\*\*# ft_transcendence README 評価要件（日本語）

42 ft_transcendence プロジェクトの README.md に求められる評価要件を日本語でまとめたものです。

---

## 基本要件

README.md は Git リポジトリのルートに配置し、プロジェクトに不慣れな人（同僚、スタッフ、採用担当者など）が、プロジェクトの内容・実行方法・参考情報を素早く把握できるようにする。

---

## 必須セクション

### 1. 先頭行（必須）

- **1行目**はイタリックで、次の文言とする：
    - `This project has been created as part of the 42 curriculum by <login1>[, <login2>[, <login3>[...]]].`
- チームの場合は、ログイン名をカンマ区切りで列挙する。

### 2. Description（説明）

- プロジェクトを明確に説明する。
- **プロジェクト名**を明記する。
- **主な機能**を記載する。
- 目的と概要を簡潔に書く。

### 3. Instructions（手順）

- コンパイル・インストール・実行に関する情報を含める。
- **前提条件**：必要なソフトウェア、ツール、バージョン、設定（例：.env の設定）を記載する。
- **実行手順**：プロジェクトを実行するためのステップバイステップの手順を書く。

### 4. Resources（参考資料）

- トピックに関連する参考文献を列挙する（ドキュメント、記事、チュートリアルなど）。
- **AI の使用**：どのタスク・プロジェクトのどの部分で AI を使用したかを具体的に記載する。

## 追加必須セクション（ft_transcendence 用）

### 5. Team Information（チーム情報）

README 先頭に記載した各メンバーについて、以下を記載する：

- **役割**：PO、PM、Tech Lead、Developer など
- **責任範囲**：担当業務の簡潔な説明

### 6. Project Management（プロジェクト管理）

- **作業の進め方**：タスク分担、ミーティングの方法など
- **プロジェクト管理ツール**：GitHub Issues、Trello など
- **連絡手段**：Discord、Slack など

### 7. Technical Stack（技術スタック）

- **フロントエンド**：使用した技術・フレームワーク
- **バックエンド**：使用した技術・フレームワーク
- **データベース**：採用した DB とその理由
- **その他の技術・ライブラリ**：重要なもの
- **主要な技術選定の理由**

### 8. Database Schema（データベーススキーマ）

- **視覚的表現または説明**：DB 構造の図や文章
- **テーブル/コレクションとリレーション**
- **主要フィールドとデータ型**

### 9. Features List（機能一覧）

- **実装済み機能の一覧**
- **各機能の担当メンバー**
- **各機能の簡単な説明**

### 10. Modules（モジュール）

- **選択したモジュール一覧**（Major と Minor）
- **ポイント計算**：Major = 2pt、Minor = 1pt
- **各モジュールの選定理由**（特にカスタム「Modules of choice」）
- **各モジュールの実装方法**
- **各モジュールの担当メンバー**

### 11. Individual Contributions（個人の貢献）

- **各メンバーの貢献の詳細**
- **担当した機能・モジュール・コンポーネント**
- **直面した課題とその解決方法**

## その他

- 使い方、既知の制限、ライセンス、クレジットなど、有用な情報があれば追加する。
- README は評価の重要な要素であり、以下の点を満たすこと：
    - 明確で整理されている
    - 必須セクションがすべて含まれている
    - 読みやすくプロフェッショナルである
    - 貢献と課題について誠実に記載している
- 不完全な README は評価に悪影響を与える可能性がある。
